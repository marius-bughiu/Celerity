Methodology

What you're looking at. Each chart plots one operation (e.g. Lookup) on one collection, with the BCL baseline and Celerity's implementation over time at 100,000 items. Y-axis is milliseconds per invocation, lower is better — click any chart for a full-size view with nanosecond precision. Numbers come from BenchmarkDotNet running in GitHub Actions on ubuntu-latest with WarmupCount=3, IterationCount=5. The full source is in src/Celerity.Benchmarks. Local Release runs use the default BenchmarkDotNet job and produce more stable numbers — this dashboard exists to catch regressions and surface trends, not as a precision instrument.

Hash function throughput

latest run
Comparing the built-in hashers. Each bar is one hasher's Hash throughput over a fixed key sample, grouped by key type and (for string) by key shape, with the framework's GetHashCode() as the baseline. Shorter is faster. This is throughput only — a fast hasher that clusters is not a win, so weigh these against the distribution metrics from HashQualityEvaluator (see the hashing docs) before picking one. Source: StringHasherBenchmark · IntegerHasherBenchmark.
